MariaDB

Was ist MariaDB?

MariaDB ist eine relationale Datenbank die sich im Featureset stark an MySQL orientiert und von einigen der originalen Entwickler von MySQL entwickelt wurde. MariaDB setzt sich vor allem dadurch ab, dass das Lizenzmodell (und die Philosophie in der Community) eine quelloffene Verfügbarkeit garantiert.

INSTALLATION

Importieren Sie den GnuPG-Signaturschlüssel mit folgendem Code

sudo apt-key adv --recv-keys --keyserver keyserver.ubuntu.com 0xcbcb082a1bb943db

MariaDB zur Datei sources.list hinzufügen. Öffnen Sie die Datei und fügen Sie den folgenden Code hinzu

sudo add-apt-repository 'deb http://ftp.osuosl.org/pub/mariadb/repo/5.5/ubuntuprecise main'

Aktualisieren Sie das System wie folgt

sudo apt-get update

Installieren Sie MariaDB mit dem untenstehenden Befehl

sudo apt-get install mariadb-server

Erstellen eines Benutzerkontos

Erstellen Sie ein neues Benutzerkonto mit folgendem Code

CREATE USER 'newusername'@'localhost' IDENTIFIED BY 'userpassword';

Dieser Code fügt der Benutzertabelle eine Zeile ohne Privilegien hinzu. Sie haben auch die Möglichkeit, einen Hash-Wert für das Passwort zu verwenden. Erteilen Sie dem Benutzer Rechte mit folgendem Code

GRANT SELECT, INSERT, UPDATE, DELETE ON database1 TO 'newusername'@'localhost';

Andere Privilegien umfassen fast alle Befehle oder Operationen, die in MariaDB möglich sind. Nachdem Sie einen Benutzer angelegt haben, führen Sie ein "FLUSH PRIVILEGES" Kommando aus, um die Berechtigungstabellen zu aktualisieren. Dies ermöglicht die Verwendung des Benutzerkontos.

Erstellen einer Datenbank

Das folgende Beispiel demonstriert die Verwendung des mysqladmin Binary bei der Erstellung einer Datenbank mit dem Namen Products

[root@host]# mysqladmin -u root -p create PRODUCTS
Enter password:******

Bei erfolgreicher Ausführung erhalten Sie folgende Ausgabe

mysql> Database PRODUCTS created successfully 
mysql> SHOW DATABASES; 
+-----------------------+ 
| Database              | 
+-----------------------+ 
| PRODUCTS              | 
+-----------------------+

Löschen einer Datenbank

DDas folgende Beispiel zeigt, wie man die mysqladmin-Binärdatei verwendet, um eine bestehende Datenbank zu löschen

[root@host]# mysqladmin -u root -p drop PRODUCTS
Enter password:******

Bei erfolgreicher Ausführung erhalten Sie folgende Ausgabe

mysql> Database PRODUCTS deleted successfully

Auswählen einer Datenbank

Bei der Auswahl einer Datenbank an der Eingabeaufforderung verwenden Sie einfach den SQL-Befehl 'use'.

[root@host]# mysql -u root -p

Enter password:******

mysql> use PRODUCTS;

Database changed

Erstellen einer Tabelle

Es folgt die allgemeine Syntax für die Tabellenerstellung

CREATE TABLE table_name (column_name column_type);

Befehl zum Erstellen einer Tabelle in der PRODUCTS-Datenbank

databaseproducts_ tbl(
   product_id INT NOT NULL AUTO_INCREMENT,
   product_name VARCHAR(100) NOT NULL,
   product_manufacturer VARCHAR(40) NOT NULL,
   submission_date DATE,
   PRIMARY KEY ( product_id )
);

Löschen einer Tabelle

Das Löschen von Tabellen ist sehr einfach, aber denken Sie daran, dass alle gelöschten Tabellen unwiederbringlich sind. Die allgemeine Syntax für das Löschen von Tabellen ist wie folgt

DROP TABLE table_name ;

Daten einfügen

DDas Einfügen von Daten in eine Tabelle erfordert den INSERT-Befehl. Die allgemeine Syntax des Befehls lautet INSERT, gefolgt von Tabellenname, Feldern und Werten.

INSERT INTO tablename (field,field2,...) VALUES (value, value2,...);

Daten auswählen

SELECT-Befehle rufen ausgewählte Zeilen ab. Sie können UNION-Anweisungen, eine Ordnungsklausel, eine LIMIT-Klausel, eine WHERE-Klausel, eine GROUP BY...HAVING-Klausel und Unterabfragen enthalten.

SELECT field, field2,... FROM table_name, table_name2,... WHERE...
